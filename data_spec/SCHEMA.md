# Synthetic data schema (final)

Generated by `scripts/generate_data.py`. **Synthetic**, **seeded** (`SEED`), with
**ground-truth labels**. Files land under `DATA_DIR` and are gitignored. The
generator models the *process physics* (nominal distributions + injected
anomalies); it deliberately does **not** know the detector's limits, so detection
isn't graded against its own cheat sheet.

## Shape
- `N_LOTS = 40` lots, `WAFERS_PER_LOT = 25` wafers each.
- Circular wafer, integer die grid of radius `WAFER_RADIUS = 8` (~201 dies/wafer).
- One CSV per lot (`LOT0000.csv` …) plus one `labels.csv`.

## Parametric records — `LOT####.csv` (one row per die per wafer)
| column | type | nominal (healthy) | notes |
|---|---|---|---|
| `lot_id` | str | — | e.g. `LOT0007` |
| `wafer_id` | int | 1..25 | wafer sequence within the lot |
| `die_x`, `die_y` | int | — | die-map coords; `x² + y² ≤ R²` |
| `Vt` | float | N(0.45, 0.02) | threshold voltage (V) |
| `Idd` | float | N(1.20, 0.05) | supply current (mA) |
| `leakage` | float | lognormal(0, 0.30) | leakage (nA): positive, right-skewed |

## Labels — `labels.csv`
| column | type | notes |
|---|---|---|
| `lot_id` | str | |
| `is_anomalous` | bool | `failure_mode != "none"` |
| `failure_mode` | str | one of the modes below |

## Lot-level process variation (makes detection non-trivial)
Every lot gets a small random **baseline offset** applied to all its dies —
`Vt += N(0, 0.008)`, `Idd += N(0, 0.025)`, leakage scaled by `exp(N(0, 0.10))`.
This models realistic lot-to-lot drift, so a naive threshold mistakes normal
variation for an anomaly (false positives) and a subtle injected shift can be
partly masked (false negatives). It is the main reason eval is honest, not 1.0.

## Injected failure modes (lot-level label; ~40% healthy) — subtle on purpose
| mode | injection | what a detector should catch |
|---|---|---|
| `none` | — (process variation only) | healthy lot |
| `idd_shift` | `Idd += 0.09` (whole lot) | mean shift on Idd |
| `tool_drift` | `Idd += 0.07·(wafer−1)/24` | gradual within-lot drift across wafers |
| `vt_tail` | ~1.5% of dies `Vt += U(0.07, 0.16)` | outlier tail on Vt |
| `edge_leakage` | edge dies (r/R > 0.80) `leakage ×= U(1.9, 2.8)` | spatial edge pattern on leakage |

## Eval (current)
Detector P/R/F1 on seed 42 ≈ **0.92 / 0.85 / 0.88** (tp=22, fp=2, fn=4, tn=14).
`tool_drift` and `vt_tail` are the hard modes; the 2 false positives are healthy
lots whose process drift trips the Idd mean-shift rule. See `scripts/eval.py`.

## Reproducibility
Mode assignment and every per-lot draw are seeded by `SEED`. Same `SEED` →
**byte-identical** CSVs (asserted in `tests/test_generate_data.py`). Regenerate on
the GPU box after `git pull`.
